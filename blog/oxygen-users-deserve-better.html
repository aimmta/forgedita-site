Oxygen-first

Why Oxygen XML users deserve better CCMS workflows.

Oxygen XML users deserve CCMS workflows that treat Oxygen as a first-class client while keeping the CCMS as the source of truth.

Oxygen users are not asking for a toy editor with a nicer login screen. They already have a serious XML environment, muscle memory, validation habits, and working authoring patterns.

A modern CCMS should not punish that. It should make Oxygen better connected: browse, open, check out, validate, preview, publish, inspect references, and check in through clean APIs.

The trick is keeping the CCMS as the source of truth while letting Oxygen remain the professional workbench. That is the line ForgeDITA is drawing.

Practical take

What this means in the real world.

Oxygen should be first-class

Not a grudging integration. Not a file-sync hack. A real client over the same APIs as everything else.

No private semantics

Oxygen convenience must not define hidden behavior that other editors cannot reproduce.

Validation belongs on both sides

Local validation helps authors work. Server validation protects repository and release integrity.

Publishing should stay server-side

The editor can request preview and publish, but the release output belongs to the CCMS runtime.
